## 1. 신규 클래스 상세

### 1-1. `Item` (Item.h / Item.cpp)

**무엇:** `Pawn`을 상속해 적과 똑같이 중력으로 떨어져 땅에 착지하고,
두 아이콘(밝은/어두운)을 번갈아 그려 깜빡이다가 수명이 끝나면 사라진다.

**핵심 멤버**
```cpp
ItemType m_type;        // 종류
int  m_lifeFrames;      // 남은 수명(프레임). 0이 되면 사망
int  m_blinkCounter;    // 현재 깜빡임 주기 경과 프레임
bool m_showBright;      // true=밝은 아이콘, false=어두운 아이콘
std::string m_brightIcon, m_darkIcon;
```

**종류 → 아이콘/색 매핑**: `switch (type)`로 분기. (Item.cpp)
- HEAL `♥`/`♡` = LIGHT_RED, STAR `★`/`☆` = LIGHT_YELLOW,
  GUN `♠`/`♤` = CYAN, KILL_ALL `◑`/`◐` = LIGHT_VIOLET.

**깜빡임 주기(점진 가속) — `CurrentBlinkPeriod()`**
```cpp
float r = (float)m_lifeFrames / (float)ITEM_LIFETIME_FRAMES; // 남은 수명 비율 0~1
period = FAST + (SLOW - FAST) * r;   // 선형 보간(lerp)
```
남은 수명이 많을 때(r≈1)는 주기가 길어(천천히 깜빡), 끝나갈수록(r→0) 주기가
짧아져(빨리 깜빡) 점점 빨라진다. 최소 1프레임으로 클램프.

**`Tick()` 순서** (Item.cpp)
1. `Pawn::Tick()` — 중력/착지(부모가 처리).
2. `--m_lifeFrames <= 0` 이면 `SetDead()`.
3. `++m_blinkCounter >= CurrentBlinkPeriod()` 이면 아이콘 교체.
   - 색(`m_renderColor`)은 안 바꾸고 **아이콘만** 바꾼다.

**`OnCollision`**: `PLAYER`면 무시(획득은 Player가 전담), 그 외는
`Pawn::OnCollision`으로 위임해 `TILE` 착지만 처리.

**`Consume()`**: 플레이어가 먹으면 호출 → `SetDead()` 표시 → 매니저가 다음
`Update`에서 제거.

### 1-2. `ItemManager` (싱글턴)

**무엇:** 아이템들을 `std::vector<std::unique_ptr<Item>>`로 들고 스폰/갱신/렌더.

**스폰 타이밍(프레임 기반)** — `TrySpawnItem(groundLength)`
```cpp
if (--m_spawnTimer > 0) return;        // 매 프레임 1 감소
m_spawnTimer = RandomSpawnDelayFrames(); // [6초,12초] 랜덤을 프레임으로
ItemType type = (ItemType)(rand() % (int)ItemType::COUNT); // 4종 균등 랜덤
int spawnX = SCREEN_WIDTH/2 - groundLength + rand()%(groundLength*2);
m_items.push_back(make_unique<Item>(type, {spawnX, 0}));
```
> **왜 프레임 기반인가:** 설정창(오버레이)이 뜨면 `GameScene::Update`가
> 호출되지 않아 이 함수도 안 불린다 → 카운터가 안 줄어 **자동으로 일시정지**.
> (적은 `GetTickCount64()` 실시간을 써서 별도 Pause/Resume이 필요했지만,
> 아이템은 프레임 카운터라 그게 필요 없다.)

**`Update()`**: 반복하며 `Tick()` 호출, `IsLeaveDeadZone() || IsDead()`면
`RemovePrevPos()`(잔상 지우기) 후 `erase`. (EnemyManager와 동일 패턴)

### 1-3. `Bullet` (독립 클래스, 중력 없음)

**무엇:** `Pawn`을 안 쓴다. `Rigidbody`에는 중력을 끄는 기능이 없어서,
`Pawn`을 쓰면 원치 않는 중력이 붙기 때문. 그래서 위치+콜라이더만 가진 가벼운
클래스로 만들어 **수평으로만** 직진시킨다.

**핵심 멤버**
```cpp
Vector2 m_pos;
int     m_dir;      // +1 오른쪽, -1 왼쪽
float   m_accumX;   // 소수 속도 누적용
bool    m_isDead;
std::unique_ptr<Collider> m_collider; // 태그 BULLET
```

**`OnCollision`**
- `ENEMY` → 적 `Kill()` + 자기 사망.
- `TILE` → **실제 겹침이 있을 때만** 사망(아래 버그2 참고).
- `PLAYER/ITEM/BULLET` → 무시.

**`Tick()` (최종본, swept collision)**
```cpp
if (m_isDead) return;
m_accumX += BULLET_SPEED;       // 3.0
int steps = (int)m_accumX;      // 보통 3
m_accumX -= (float)steps;
for (int i = 0; i < steps; ++i) // 한 칸씩 전진
{
    m_pos.x += m_dir;
    if (화면 밖) { m_isDead = true; return; }
    Collider* hit = ColliderManager::GetInst()
        ->FindOverlappingTag(m_collider.get(), ColliderTag::ENEMY);
    if (hit) { 적 Kill(); m_isDead = true; return; } // 칸마다 검사
}
```
> 왜 한 칸씩 도느냐는 **버그2(tunneling)** 절에서 설명.

`erase`. `ItemManager`와 같은 구조.

## 2. 수정한 기존 코드

| 파일 | 변경 |
|---|---|
| `Collider.h` | `enum class ColliderTag { TILE, PLAYER, ENEMY, SPIKE, ITEM, BULLET };` |
| `Defines.h` | 아이템 상수 블록 추가(6절 표 참고) |
| `EnemyManager` | `KillAll()` — 모든 적을 순회하며 `Kill()` |
| `Player` | 아이템 획득/효과/틴트/별 렌더 (3절에서 상세) |
| `GameScene.cpp` | `Init/Update/Render/Release`에 두 매니저 호출 배선 |
| `Core.cpp` | `item_pickup`/`gun_fire` 사운드 로드, `~Core`에서 두 싱글턴 파괴 |

**`Core::~Core` 파괴 순서(중요):** `Scene → Enemy/Item/Bullet 매니저 →
ColliderManager` 순. 아이템/총알이 콜라이더를 들고 있으므로, 그것들을 먼저
없애야 `ColliderManager`가 안전하게 파괴된다.

**`GameScene` 호출 순서**
- Update: 적 스폰 → 아이템 스폰 → 플레이어 Tick → 적/아이템/총알 Update
- Render: 맵 → 적 → 아이템 → 총알 → 플레이어 (플레이어가 맨 위에 그려짐)

## 3. Player 효과 통합 (가장 큰 변경)

모든 효과 상태는 **프레임 카운터**라 오버레이 중 자동 정지된다.

**추가 멤버**
```cpp
int m_starTimer, m_gunTimer;   // 남은 프레임(>0이면 효과 켜짐)
int m_gunFireCounter;          // 다음 발사까지 남은 프레임
int m_facing = 1;              // 마지막 이동 방향(+1/-1), 기본 오른쪽
ItemType m_lastPersistentPick; // 가장 최근에 먹은 지속 효과(틴트 우선순위용)
```

**획득 — `OnCollision`의 `ITEM` 분기**
```cpp
Item* item = static_cast<Item*>(other->GetOwner());
if (!item || item->IsDead()) return;
ApplyItem(item->GetType());
item->Consume();
SOUND->Play("item_pickup");
```
무적 가드가 없으므로 **무적 중에도 획득**된다.

**효과 적용 — `ApplyItem(type)` (switch)**
- HEAL: `if (m_hp < m_maxHp) ++m_hp;`
- STAR: `m_starTimer = 300(5초)`, 아이콘을 `"★"`로, 최근픽=STAR.
- GUN: `m_gunTimer = 360(6초)`, `m_gunFireCounter = 1`(다음 프레임 첫 발), 최근픽=GUN.
- KILL_ALL: `EnemyManager::KillAll()` + CRIT 차단형 흔들림 **2배** 세기.

**별 즉사 우선순위 — `OnCollision`의 `ENEMY` 분기 순서**
```
적 null/이미죽음 검사
→ 별이 켜져 있으면: 적 Kill() 후 return (플레이어 무피해)  ← 무적/낙하검사보다 먼저
→ 무적이면 return
→ 낙하 중 적이면 즉사
→ 일반 피격(ResolveEnemyHit)
```

**매 프레임 갱신 — `UpdateItemEffects()`** (Tick의 **두 경로 모두**에서 호출)
- `m_facing`: 속도 부호로 갱신(정지 시 직전 값 유지).
- 별 타이머: 0이 되면 아이콘 `"●"`로 복귀.
- 총 타이머: 감소하며 `--m_gunFireCounter <= 0`이면
  `BulletManager::Spawn(m_pos, m_facing)` + `gun_fire` 사운드, 카운터 리셋.

**모습(아이콘+색) 결정 — `UpdateAppearance()` / `CurrentTintType()`**
매 프레임 Tick에서 호출되어 `m_renderIcon`과 `m_renderColor`를 함께 정한다.
```cpp
// 종료 임박(마지막 1초) 동안 효과/평상 모습을 번갈아 보여준다.
int warnT = (별 워닝중 ? m_starTimer : 총 워닝중 ? m_gunTimer : 0);
bool showNormal = warnT>0 && ((warnT / PLAYER_BLINK_INTERVAL) % 2 == 1);

if (효과 없음 || showNormal) {            // 평상 모습
    m_renderIcon  = "●";
    m_renderColor = Enemy::ColorForSpeed(ColorSpeed()); // 속도 색
} else {                                  // 효과 모습
    m_renderColor = Item::ColorForType(CurrentTintType()); // 최근 효과 색(틴트)
    m_renderIcon  = starOn ? "★" : "●";   // 별이면 ★, 총만이면 ●
}
```
- 평소: 별/총이 없으면 `●` + 속도 색.
- 별/총 지속 중: 틴트 색(별이면 아이콘도 ★).
- **종료 1초 전**: 효과 모습 ↔ 평상 모습을 프레임마다 번갈아 그려 깜빡인다.
  (별=★/틴트 ↔ ●/속도색, 총=●/틴트 ↔ ●/속도색). 기존의 "숨김" 방식 대신
  두 상태를 교대하는 방식이다.

중요: 크리티컬 판정은 `m_renderColor`가 아니라 계산값 `ColorSpeed()`로 하므로,
표시색만 바뀌고 **게임 판정은 그대로** 동작한다.

> `Render()`에는 더 이상 별 전용 깜빡임이 없다(무적 깜빡임만 남음). 깜빡임은
> `UpdateAppearance`가 아이콘/색을 교대로 세팅하는 것으로 표현된다.

## 4. 핵심 메커니즘 (how)

### 싱글턴(Singleton)
```cpp
static ItemManager* GetInst() { if (!m_inst) m_inst = new ItemManager(); return m_inst; }
static void DestroyInst() { SAFE_DELETE(m_inst); }
static ItemManager* m_inst; // .cpp에서 nullptr로 정의
```
생성자를 `private`로 막아 인스턴스가 하나만 존재하게 한다. 어디서든
`ItemManager::GetInst()->...`로 접근.

### 콜라이더 & 충돌 콜백
- 각 콜라이더는 생성 시 `ColliderManager`에 등록, 소멸자(`~Collider`)에서
  자동 해제. 그래서 `unique_ptr<Collider>`만 들고 있으면 정리가 자동.
- `ColliderManager::Update()`가 모든 쌍을 `Overlaps`로 검사하고, 겹치면
  양쪽의 `OnCollision` 콜백(람다로 등록)을 호출.

### `Overlaps`는 경계 포함(inclusive)
```cpp
GetLeft() <= other.GetRight() && GetRight() >= other.GetLeft() && (세로도 동일)
```
`<=`, `>=`라서 **딱 맞닿기만 해도(겹침 0)** 충돌로 친다. 이 성질이 아래 두
버그의 원인이 됐다.

## 5. 작업 중 발견·수정한 버그 2개 (디버깅 사례)

### 버그1 — 총알이 바닥에 닿자마자 사라짐
- 총알은 플레이어 줄(`groundY-1`)에서 생성된다. 총알 아랫면(`groundY`)이
  바닥 콜라이더 윗면(`groundY`)과 **경계로 맞닿음** → `Overlaps`가 inclusive라
  "겹쳤다"고 판정 → 옛 코드는 `TILE`이면 무조건 사망 → 1프레임 만에 소멸.
- **수정:** `TILE` 충돌을 **실제 겹침(>0)일 때만** 사망하도록(아래). 바닥은
  겹침 0이라 스쳐 지나가고, 진짜 파고든 벽만 막는다. (Pawn의 충돌 처리와 동일 방식)
```cpp
int overlapX = min(내right, 상대right) - max(내left, 상대left);
int overlapY = min(내bottom, 상대bottom) - max(내top, 상대top);
if (overlapX > 0 && overlapY > 0) m_isDead = true;
```

### 버그2 — 적이 총알에 안 맞음 (tunneling)
- 총알이 매 프레임 `3칸`을 **한 번에 점프**하고, 충돌은 이동 후 최종 위치에서만
  검사. 적도 다가오므로 상대 접근속도 ≈ 3 + 1.9 ≈ **5칸/프레임**.
- 적-총알 겹침 윈도우는 3칸뿐이라, 5칸씩 좁혀지면 그 3칸을 **건너뛰어** 충돌이
  안 잡힌다(고속 발사체의 전형적 tunneling).
- **수정:** `Tick()`에서 **한 칸씩 전진하며 칸마다** `FindOverlappingTag(ENEMY)`로
  검사 → 어떤 속도라도 적을 건너뛰지 않음. 속도(3칸/프레임)는 유지.

> 두 버그 모두 근본 원인이 `Overlaps`의 경계 포함 + 1프레임 단위 이산 충돌이라는
> 공통점이 있다.

## 7. 사용한 C++ 문법·표준 함수 (채점 대비 빠른 색인)

| 요소 | 어디서 | 한 줄 설명 |
|---|---|---|
| `enum class` | `ItemType`, `ColliderTag` | 스코프 있는 열거형(암묵 int 변환 안 됨 → 명시 캐스팅) |
| `static` 멤버 + private 생성자 | 매니저들 | 싱글턴 구현 |
| `std::unique_ptr` / `std::make_unique` | 아이템·총알·콜라이더 | 소유권 단독, 자동 해제 |
| `std::move` | `push_back(move(...))` | 소유권 이전(복사 대신) |
| `std::vector` + 반복 중 `erase` | 매니저 Update | `it = v.erase(it); continue;` 안전 삭제 패턴 |
| `std::function` + 람다 | 콜라이더 콜백 | `SetOnCollision([this](Collider* c){...})` |
| `std::min`/`std::max` | 겹침 계산 | AABB 겹침 폭 = min(우변) - max(좌변) |
| `switch` | `ApplyItem`, 아이콘/색 매핑 | 종류별 분기 |
| `static_cast` | `GetOwner()` → `Enemy*`/`Item*` | `void*` 다운캐스트 |
| `(int)` 캐스팅 + float 누적 | 총알/아이템 이동, 보간 | 소수 속도를 정수 칸으로 |
| `const` 멤버 함수 / `mutable` | `Render() const` | 논리적 const 안에서 캐시 멤버만 변경 |
| 상속/가상함수 오버라이드 | `Item : Pawn`, `Tick/OnCollision` | 부모 동작 재사용 + 확장 |
| 선형 보간(lerp) | 깜빡임 주기 | `A + (B-A)*t` |
